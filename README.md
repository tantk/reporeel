# RepoReel

**An AI agent that turns any GitHub repo into a narrated video walkthrough — Claude orchestrates HeyGen Avatar IV and Hyperframes from a single slash command.**

> **HeyGen Hackathon · Agent Track · May 2026**

## HeyGen features used

| Feature | Endpoint / framework | Role in the pipeline |
|---|---|---|
| **Avatar IV** | `POST /v3/videos` with Photo Avatar `f20cdc89e0ec4b61bbe453d73019a997` (Madison) | Renders the 3 spoken scenes — the presenter in the bottom-right of every output video |
| **Hyperframes** | `@hyperframes` (open-source framework) | Composes the slide deck around the avatar and stitches everything into one deterministic MP4 via headless Chrome + FFmpeg |

Two HeyGen products are load-bearing nodes in the agent pipeline. Remove either and the agent breaks.

## The agent pipeline

```
  USER: /reporeel github.com/owner/repo
       │
       ▼
  ┌────────────────────────────────────────────────────────────┐
  │  Claude Code session (the orchestrator)                    │
  │                                                            │
  │  1. WebFetch  ──── GitHub REST API + raw README.md         │
  │  2. Reason    ──── Claude plans 3 scenes (35-45 words/ea)  │
  │  3. Approval  ──── show plan in chat, wait for `y`         │
  │  4. Bash      ──── render-scenes.sh                        │
  │                       └─→ HeyGen Avatar IV /v3/videos × 3  │
  │                       └─→ poll + download MP4s             │
  │  5. Write     ──── Hyperframes composition (HTML)          │
  │  6. Bash      ──── hyperframes render                      │
  │                       └─→ headless Chrome + FFmpeg         │
  └────────────────────────────────────────────────────────────┘
       │
       ▼
  outputs/<owner>--<repo>/final.mp4   (shareable, single file)
```

### The stack

| Layer | Tool | Role |
|---|---|---|
| Driver | **Claude Code** (Sonnet / Opus) | Reads the spec in `SKILL.md` and runs the workflow step-by-step |
| Input | **WebFetch** (Claude built-in) | Pulls GitHub repo metadata + README |
| Reasoning | **Claude** (in-session, no external LLM call) | Writes the 3-scene narration |
| Avatar render | **HeyGen Avatar IV** (`POST /v3/videos`) | Generates 3 spoken video scenes with Madison Photo Avatar |
| Composition | **Hyperframes** (open-source, `@hyperframes`) | Composes HTML slide deck + stitches scenes into one MP4 |
| Output | Local filesystem | `outputs/<repo>/final.mp4` — single shareable file |

### The edge

- **The whole agent is a 200-line markdown file.** Spec lives in [`skills/reporeel/SKILL.md`](skills/reporeel/SKILL.md). Anyone can read, audit, or fork the orchestration in plain English.
- **No external LLM API key** required. Claude drives the workflow inside the user's existing session.
- **Two HeyGen products in one pipeline**: Avatar IV (paid REST API) for the presenter; Hyperframes (open-source framework) for the composition. Both products are load-bearing — neither is a swap-in dependency.
- **Self-demonstrating output**: the videos above were rendered by the same pipeline they describe. The Hyperframes-rendered slides explain what Hyperframes does. The Avatar IV presenter explains how the agent uses Avatar IV.

## Usage

```
/reporeel https://github.com/owner/repo
```

Claude prints a 3-scene plan, asks for approval, then runs the pipeline. Total wall time: ~5 minutes (HeyGen renders + Hyperframes encode).

## Setup

See **[INSTALL.md](./INSTALL.md)** for the full 5-minute setup (Windows / macOS / Linux, prerequisites, troubleshooting).

Short version:

1. `git clone https://github.com/tantk/reporeel C:\dev\reporeel`
2. `cp .env.example .env` + add your HeyGen API key
3. `npm i -g hyperframes`
4. Symlink `skills/reporeel/` into `~/.claude/skills/`
5. Restart Claude Code, type `/reporeel <github-url>`

## Files

| Path | Role |
|------|------|
| `skills/reporeel/SKILL.md` | The agent spec — the entire orchestration in markdown |
| `scripts/render-scenes.sh` | HeyGen Avatar IV node (REST + polling + caching) |
| `hyperframes-build/index.html` | Hyperframes composition node — slide deck around the presenter |
| `reference/default-avatar.json` | Photo Avatar selection (Madison Photo Avatar) |
| `outputs/<owner>--<repo>/` | Per-run artifacts: plan.json + scenes/*.mp4 + final.mp4 |
| `docs/DEMO.md` | Demo Day script + submission writeup |
