# RepoReel

**Turn any GitHub repo into a presentable video walkthrough.** A Claude Code skill that orchestrates HeyGen Avatar IV + Hyperframes.

## 🎬 Demo videos (HeyGen Hackathon · May 2026)

- **RepoReel on RepoReel** (the meta video) → https://youtu.be/DBeDU9EDAPs
- **RepoReel on `heygen-com/hyperframes`** → https://youtu.be/OrBq-vtvimw

Both videos were autogenerated by running the skill on the respective repos. The avatar presenter is HeyGen Avatar IV. The slide deck beside it is composed by Hyperframes. The narration was written by Claude during the skill run.

## Usage

In a Claude Code session:

```
/reporeel https://github.com/owner/repo
```

Claude fetches the repo, drafts a 3-scene plan, shows it for approval, then renders the avatar scenes via HeyGen and assembles the final MP4 via Hyperframes.

## Setup

1. Copy `.env.example` to `.env` and add your HeyGen API key.
2. Install the Hyperframes CLI: `npm i -g hyperframes` (or use `npx`).
3. Symlink or copy `skills/reporeel/` into your Claude Code skills directory.

## Architecture

```
  USER: /reporeel github.com/owner/repo
       │
       ▼
  ┌────────────────────────────────────────────────────┐
  │  Claude Code session                               │
  │                                                    │
  │  1. WebFetch — README + GitHub metadata            │
  │  2. Plan — Claude writes 3 narration scripts       │
  │  3. Approval gate — show user, wait for `y`        │
  │  4. HeyGen Avatar IV — POST /v3/videos × 3         │
  │  5. Hyperframes — HTML composition → MP4           │
  └────────────────────────────────────────────────────┘
       │
       ▼
  outputs/<owner>--<repo>/final.mp4   (shareable, single file)
```

- **No backend.** No Anthropic API key needed (Claude is the agent driver).
- **HeyGen Avatar IV** renders the spoken presenter (Madison Photo Avatar by default).
- **Hyperframes** is HeyGen's open-source HTML-to-MP4 framework — it does the slide composition, animated transitions, scene labels, and final encode via headless Chrome + FFmpeg.

## Why both Product Track and Agent Track

- **Product:** Ships as a single markdown skill on the Anthropic Skills marketplace. Zero infra cost on our side. Every OSS maintainer, YC founder, and dev influencer wants this.
- **Agent:** The skill IS the pipeline — Claude orchestrates WebFetch → reasoning → REST → Hyperframes engine in plain markdown. The whole agent stack is auditable in 200 lines.
