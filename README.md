# RepoReel

**An AI agent that turns any GitHub repo into a narrated video walkthrough вҖ” Claude orchestrates HeyGen Avatar IV and Hyperframes from a single slash command.**

> **HeyGen Hackathon В· Agent Track В· May 2026**

## HeyGen features used

| Feature | How we invoke it | Role in the pipeline |
|---|---|---|
| **Avatar IV** | Official **HeyGen CLI** ([`heygen-com/heygen-cli`](https://github.com/heygen-com/heygen-cli)) вҖ” `heygen video create --wait` with Photo Avatar `f20cdc89e0ec4b61bbe453d73019a997` (Madison) | Renders the 3 spoken scenes вҖ” the presenter in the bottom-right of every output video |
| **Hyperframes** | Open-source `@hyperframes` framework вҖ” `hyperframes render` | Composes the slide deck around the avatar and stitches everything into one deterministic MP4 via headless Chrome + FFmpeg |

Two HeyGen products are load-bearing nodes in the agent pipeline. Both are consumed through their **official, agent-first tooling** (CLI + framework) rather than reimplementing вҖ” so RepoReel is a vertical app on top of the HeyGen platform, not a duplicate of it.

## Differentiation from related work

The HeyGen ecosystem already includes:

- **`heygen-com/skills`** вҖ” `heygen-avatar`, `heygen-video`, `heygen-translate` вҖ” the official platform skills for generic avatar/video/translate flows.
- **`heygen-com/website-to-hyperframes`** вҖ” captures a website's UI as `<video>` clips and composes them via Hyperframes; uses ElevenLabs TTS (not Avatar IV); not Claude-orchestrated.
- **`joshua-heygen/viz-pack`** вҖ” Claude skills that turn content into static visuals (diagrams, infographics, explainers, dashboards).

RepoReel sits in the gap: **GitHub repo as input вҶ’ Avatar IV-narrated presenter + Hyperframes-composed slides вҶ’ single MP4**, with Claude driving the whole orchestration from one slash command. It's a vertical app on top of the platform, not a replacement for any of the existing skills.

**Step 5 is where the design lives.** Unlike a slot-filling template, Claude designs the Hyperframes composition fresh per repo вҖ” a graph database gets a node diagram, a CLI tool gets a terminal mock, a web framework gets a code block + browser preview. The brand constants (avatar in bottom-right, dark theme, 1920Г—1080) stay fixed; everything else is custom.

Design instructions live in [`skills/reporeel/references/hyperframes-patterns.md`](skills/reporeel/references/hyperframes-patterns.md) вҖ” required reading for Claude before it writes the composition.

### The stack

| Layer | Tool | Role |
|---|---|---|
| Driver | **Claude Code** (Sonnet / Opus) | Reads the spec in `SKILL.md` and runs the workflow step-by-step |
| Input | **WebFetch** (Claude built-in) | Pulls GitHub repo metadata + README |
| Reasoning | **Claude** (in-session, no external LLM call) | Writes the 3-scene narration |
| Avatar render | **HeyGen Avatar IV** (`POST /v3/videos`) | Generates 3 spoken video scenes with Madison Photo Avatar |
| Composition | **Hyperframes** (open-source, `@hyperframes`) | Composes HTML slide deck + stitches scenes into one MP4 |
| Output | Local filesystem | `outputs/<repo>/final.mp4` вҖ” single shareable file |

### The edge

- **The whole agent is a 200-line markdown file.** Spec lives in [`skills/reporeel/SKILL.md`](skills/reporeel/SKILL.md). Anyone can read, audit, or fork the orchestration in plain English.
- **No external LLM API key** required. Claude drives the workflow inside the user's existing session.
- **Two HeyGen products in one pipeline**: Avatar IV (paid REST API) for the presenter; Hyperframes (open-source framework) for the composition. Both products are load-bearing вҖ” neither is a swap-in dependency.
- **Self-demonstrating output**: the videos above were rendered by the same pipeline they describe. The Hyperframes-rendered slides explain what Hyperframes does. The Avatar IV presenter explains how the agent uses Avatar IV.

## Usage

```
/reporeel https://github.com/owner/repo
```

Claude prints a 3-scene plan, asks for approval, then runs the pipeline. Total wall time: ~5 minutes (HeyGen renders + Hyperframes encode).

## Setup

See **[INSTALL.md](./INSTALL.md)** for the full 5-minute setup (Windows / macOS / Linux, prerequisites, troubleshooting).

Short version:

1. `git clone https://github.com/tantk/reporeel C:\dev\reporeel`
2. `cp .env.example .env` + add your HeyGen API key
3. `npm i -g hyperframes`
4. Symlink `skills/reporeel/` into `~/.claude/skills/`
5. Restart Claude Code, type `/reporeel <github-url>`
