Agent pipeline

The whole orchestration is 200 lines of markdown

Claude reads SKILL.md and runs the workflow step-by-step. No backend, no Anthropic API key, no hidden state.

USER: /reporeel github.com/owner/repo
    │
    ▼
Claude Code session (the orchestrator)
  1. WebFetch  ──── GitHub REST API + raw README.md
  2. Reason    ──── Claude plans 3 scenes (35–45 words each)
  3. Approval  ──── show plan in chat, wait for `y`
  4. Bash      ──── render-scenes.sh → HeyGen Avatar IV /v3/videos × 3
  5. Write     ──── Hyperframes composition (HTML)
  6. Bash      ──── Hyperframes render → headless Chrome + FFmpeg
    │
    ▼
outputs/<owner>--<repo>/final.mp4
HeyGen features used

Two HeyGen products. Both load-bearing.

Avatar IV is the paid REST API node. Hyperframes is the open-source local-render node. Remove either and the agent breaks.

FeatureEndpoint / frameworkRole in the pipeline
Avatar IV POST /v3/videos with Photo Avatar Madison Renders the 3 spoken scenes — the presenter you see in the bottom-right of every output video.
Hyperframes @hyperframes (open-source framework) Composes the slide deck around the avatar and stitches everything into one deterministic MP4 via headless Chrome + FFmpeg.
Install

Try it yourself (5 minutes)

$ git clone https://github.com/tantk/reporeel C:\dev\reporeel
$ cd C:\dev\reporeel
$ cp .env.example .env  # then add your HeyGen API key
$ npm i -g hyperframes

# Link the skill into Claude Code
$ ln -s "$PWD/skills/reporeel" ~/.claude/skills/reporeel

# Restart Claude Code, then in any session:
> /reporeel https://github.com/anthropics/claude-code
